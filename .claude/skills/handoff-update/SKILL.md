# Handoff Update

Use this skill at natural stopping points after meaningful work.

## Goal

Keep `HANDOFF.md` accurate enough for a cold resume without turning it into a transcript.

## Update Checklist

Refresh these sections:

- current status / phase
- what was done
- what is next
- open decisions or blockers

Also capture:

- important schema or architecture decisions made this session
- verification results
- any repo state the next session should not rediscover from scratch

## Rules

- Summarize outcomes, not every command.
- Prefer durable facts over temporary reasoning.
- Do not update `HANDOFF.md` mid-task unless explicitly asked.
